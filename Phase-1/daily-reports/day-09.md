# Day 09 – 2026-02-24

## 📌 Focus of the Day

- Mastering Gradients, Shorthand Backgrounds, and Transitions in CSS.

## 📚 What I Learned

### Gradients

Practiced creating Linear Gradients using direction keywords (`to right`, `to bottom`, etc.) and angle degrees.
Learned to create faded gradients with multiple color stops and percentages.
Explored `cssgradient.io` to visualize and export complex gradient codes.

### Background Shorthand

Combined `color, image, position, size, repeat, origin, clip`, and `attachment` into one background property.

### Browser Compatibility

- Used `caniuse.com` to test and verify CSS property support (e.g., `border-radius` prefixes).

### Transitions

Applied `transition: all 500ms ease-in 50ms;` for smooth hover animations.
Understood `transition-property, transition-duration, transition-timing-function, and transition-delay`.
Experimented with `cubic-bezier.com` for custom easing curves.

### Projects

Created new folders: `train3.html` & `train4.html` for focused gradient and transition practices.
Revisited the Filimo project to integrate new visual effects.

## 🧠 Key Takeaways

- Gradient angles (`0deg, 90deg, etc.`) control fade direction precisely.
- The all-in-one background shorthand keeps code cleaner and compact.
- `ease-in + hover` transitions greatly enhance UI smoothness.
- Testing prefixes on `caniuse.com` ensures cross-browser stability.
